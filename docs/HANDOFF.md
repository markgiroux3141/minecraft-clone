## Where the project stands

M0–M5 are done and verified (built, run, screenshot-checked):
- Engine (`engine/src/vox/`, namespace `vox`): fixed-timestep app loop
  (20 TPS + interpolated render), GLFW window/input, GL 4.6 renderer facade
  with DSA abstractions (Shader, Buffer/VertexArray, Texture2D,
  Texture2DArray, PerspectiveCamera, Frustum), spdlog logging, asset-root
  resolution (`vox::assets`), `vox::ThreadPool` (std::jthread workers,
  FIFO queue of `move_only_function`; queued-but-unstarted jobs are
  discarded on shutdown).
- Game (`game/src/`, namespace `vc`): data-driven `BlockRegistry` (air=0,
  stone, dirt, grass), 16^3 `Chunk` (YZX order), greedy `ChunkMesher` with
  baked AO (see below), `World` streaming manager (threaded, see below),
  FastNoiseLite heightmap `TerrainGenerator` (seed 1337 in GameApp),
  `FlyCamera` (RMB look, WASD + Space/Shift, Ctrl boost).
- Block textures: GL texture array (NOT an atlas — no mip bleeding, UVs can
  tile for greedy meshing). Layers generated by `scripts/gen_textures.py`
  into `assets/textures/atlas.png`; layer order must match
  `blocks::RegisterDefaults()` in `game/src/world/Block.cpp`.

## M4 threaded pipeline (how it works)

- `World` owns a `vox::ThreadPool`; generation and meshing run as jobs.
  Workers produce CPU-side data only (`shared_ptr<const Chunk>`,
  `ChunkMesh`) and push it to mutex-guarded completion vectors; `Update()`
  drains them on the main thread, which exclusively owns the chunk map and
  all GL calls.
- `ChunkEntry` holds `shared_ptr<const Chunk>` + a `ChunkState`
  (Generating → NeedsMesh → Meshing → Ready). Chunks are IMMUTABLE after
  generation, so meshing jobs take a `ChunkSnapshot` (shared_ptrs to the
  full 3x3x3 neighborhood — AO needs edge/corner chunks — built on the
  main thread): no copying, no locks on block data.
  `ChunkMesher::Build(snapshot)` never touches `World`; meshing gates on
  all 26 neighbors having data (`NeighborsHaveData`).
- Stale results (chunk unloaded mid-job) are dropped via the state check
  in `DrainCompletedJobs`; re-requested coords accepting an old result is
  harmless because generation is deterministic. **M6 block edits will need
  remesh versioning / copy-on-write — see comments in `World.h`/`.cpp`.**
- Job submission is capped at `workers * 4` in flight so the queue stays
  short and re-sorts nearest-first as the camera moves; mesh jobs are
  submitted before gen jobs. View radius now 12 (data radius 13 → 2916
  chunks ≈ 24 MB of block data); world height still 4 chunks (terrain
  tops out at y≈44, so raising height waits until terrain gets taller).
- `World` member order matters: the pool is declared LAST so its
  destructor joins workers before the queues/map/generator die.

## Known issues / decisions in flight

- One-time NVIDIA warning at startup: "Vertex shader in program 3 is being
  recompiled based on GL state" — harmless, not yet investigated.
- GPU uploads are not budgeted per frame; bursts are bounded by the
  in-flight job cap (~60 on a 16-thread machine). Fine so far; budget
  uploads if frame spikes show up at higher radii.

## M5 greedy meshing + AO (how it works)

All in `ChunkMesher.cpp`, pure functions of the snapshot:
- The worker first flattens the snapshot into a padded 18^3 volume
  (block ids + opacity) for O(1) lookups; ~18 KB on the worker's stack.
- Per face direction (FaceBasis table: slice axis, sign, u/v axes chosen
  so cross(u,v) = normal → CCW winding; cull face is ON, wrong winding
  shows as invisible faces), per 16x16 slice: build a mask keyed on
  (texture layer, 4 corner AO values), then standard greedy rectangle
  merge. Identical-key merging keeps AO gradients continuous.
- AO is the classic 3-neighbor corner term (0..3, baked as 0..1 float in
  `ChunkVertex::ao`); quads split along the brighter diagonal to avoid
  the dark-cross artifact. `chunk.frag` applies `mix(0.4, 1.0, v_ao)`.
- UVs span (w, h) tiles; the texture array uses GL_REPEAT so quads tile.
  Result: ~0.39M triangles for the whole radius-12 world (~10x less than
  culled meshing). Title bar shows the loaded-triangle total.
- Only opaque blocks are meshed; transparent block types (water, leaves)
  will need their own mesh pass when they exist.
